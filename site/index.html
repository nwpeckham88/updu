No. 01 — Coverage

A small surface area, with broad enough reach.

updu doesn't try to be a platform. It's one tool that watches the things users feel, the things underneath, and the operator workflow around them.

  1. I

    The stack users feel

    External surfaces first — the things a customer or teammate notices when something breaks.

    • HTTP
    • JSON API
    • WebSocket
    • TCP
    • Ping
    • DNS
    • SSL
  2. II

    The dependencies underneath

    Database, cache, mail, and remote-access checks in the same tool — instead of three more dashboards.

    • SSH
    • UDP
    • SMTP
    • Redis
    • PostgreSQL
    • MySQL
    • MongoDB
  3. III

    The operator loop too

    Incidents, maintenance windows, status pages, and alerting all live in the same deployable.

    • Push / Heartbeat
    • Incidents
    • Maintenance
    • Status pages
    • Webhook · Slack · Discord
    • Email · Ntfy
    • GitOps YAML

Current release v0.5.1 Release notes ↗

Linux AMD64 is shown by default. Switch architecture, or enable the OIDC build if you need single sign-on.

Binary

Enable this only if you want the release that includes OpenID Connect. 

curl -fsSLO https://github.com/nwpeckham88/updu/releases/download/v0.5.1/updu-linux-amd64 \
  && chmod +x updu-linux-amd64 \
  && ./updu-linux-amd64

Paste it, let the binary start, then open http://localhost:3000.
