№ 03

A small surface,
with broad reach.

Three concentric concerns. Watched by one process, written to one database, exposed through one UI.

  1. I

    Surfaces users feel

    External endpoints first — what a customer notices when something breaks.

    • HTTP
    • JSON API
    • WebSocket
    • TCP
    • Ping
    • DNS
    • SSL
  2. II

    Dependencies underneath

    Database, cache, mail, remote access — same tool, not three more dashboards.

    • SSH
    • UDP
    • SMTP
    • Redis
    • PostgreSQL
    • MySQL
    • MongoDB
  3. III

    The operator loop

    Incidents, maintenance windows, public status pages, alerting — all in the same deployable.

    • Push
    • Heartbeat
    • Incidents
    • Maintenance
    • Status pages
    • Webhook · Slack · Discord
    • Email · Ntfy

Advanced monitors extend that surface area further: HTTPS combines response and TLS policy, Composite rolls up existing monitors with quorum logic, Transaction follows multi-step HTTP flows, and DNS+HTTP verifies DNS-to-origin consistency.

№ 04

One process.
Composed of parts that fit.

UI, API, scheduler, auth, and storage are not microservices. They are rooms in the same house.

  • Single binary. No Docker required, no sidecar, no cluster.
  • Embedded SvelteKit SPA served from the same process as the API.
  • SQLite with WAL. The database is a file. Backup is a copy.
  • SSE live updates. No polling. No client-side intervals.
  • Prometheus & /healthz. Drop into any metrics or probe stack.
№ 05

Pick a method.
Paste one line.

Three ways in, equally short. Pick whichever your environment already speaks.

Current release v0.5.1 notes ↗

A

Binary

Linux AMD64 default. Open :3000 when it's running.

VER=v0.5.1
curl -LO https://github.com/nwpeckham88/updu/releases/download/$VER/updu-linux-amd64
chmod +x updu-linux-amd64 && ./updu-linux-amd64
B

Docker

For hosts that already speak containers.

docker run -d --name updu \
  -p 3000:3000 -v updu-data:/data \
  -e UPDU_AUTH_SECRET=$(openssl rand -hex 32) \
  ghcr.io/nwpeckham88/updu:latest
C

systemd

Drop a hardened unit. Survive reboots.

sudo ./updu install
sudo systemctl start updu
